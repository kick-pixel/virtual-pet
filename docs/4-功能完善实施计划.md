```java
private Map<String, Object> oauth2Providers; // Spring 按名注入 GoogleOAuth2Provider 等

@GetMapping("/oauth/{provider}/url")
public AjaxResult getOAuthUrl(@PathVariable String provider,
                               @RequestParam(required = false) String redirectUri) {
    // 1. 根据 provider 获取对应的 OAuth2Provider（google/microsoft/twitter）
    // 2. 生成随机 state，Redis 存储（key: "oauth:state:{state}"，TTL=10min）
    // 3. 调用 provider.getAuthorizationUrl(state)
    //    注意：如果前端传了 redirectUri，需要覆盖配置中的 redirect-uri
    // 4. 返回完整授权 URL
    return AjaxResult.success(Map.of("url", authUrl));
}

@PostMapping("/oauth/{provider}/callback")
public AjaxResult oauthCallback(@PathVariable String provider,
                                 @RequestBody Map<String, String> body) {
    String code = body.get("code");
    String state = body.get("state");
    // 1. Redis 验证 state（防 CSRF），验证后删除
    // 2. 调用 provider.handleCallback(code) 获取 OAuth2UserInfo
    // 3. 调用 virtualUserService.registerOrLoginByOAuth(
    //        provider, userInfo.getId(), userInfo.getEmail(),
    //        userInfo.getName(), userInfo.getAvatar())
    // 4. 生成 JWT token（复用 VirtualTokenService + createLoginResult）
    // 5. 记录登录（recordLoginSuccess）
    // 6. 返回 { token, userId, username, email, avatarUrl }
    return AjaxResult.success(createLoginResult(user, token));
```

OAuth 流程采用**弹窗模式**（与钱包登录一致的 UX）：

#### `OAuthCallback.vue` 实现

```vue
<script setup lang="ts">
import { onMounted, ref } from 'vue'
import { oauthCallback } from '@/api/auth'
import { useRoute } from 'vue-router'

const route = useRoute()
const status = ref<'processing' | 'success' | 'error'>('processing')
const errorMsg = ref('')

onMounted(async () => {
  const code = route.query.code as string
  const state = route.query.state as string
  const provider = route.query.provider as string || 'google'

  if (!code) {
    status.value = 'error'
    errorMsg.value = 'Authorization code not found'
    return
  }

  try {
    const res: any = await oauthCallback(provider, code, state)
    status.value = 'success'

    // 通知父窗口（Header.vue 的弹窗）
    if (window.opener) {
      window.opener.postMessage(
        { type: 'oauth-callback', data: res.data },
        window.location.origin
      )
      setTimeout(() => window.close(), 500)
    } else {
      // 非弹窗模式：直接存 token 并跳转首页
      localStorage.setItem('virtual_token', res.data.token)
      window.location.href = '/'
    }
  } catch (e: any) {
    status.value = 'error'
    errorMsg.value = e.message || 'Login failed'
  }
})
</script>

<template>
  <div style="display: flex; align-items: center; justify-content: center;
              min-height: 100vh; background: #f5f5f5;">
    <div v-if="status === 'processing'" style="text-align: center;">
      <a-spin size="large" />
      <p style="margin-top: 16px; color: #666;">Processing login...</p>
    </div>
    <div v-else-if="status === 'error'" style="text-align: center;">
      <p style="color: #ef4444; font-size: 16px;">{{ errorMsg }}</p>
      <a-button @click="window.close()">Close</a-button>
    </div>
  </div>
</template>
```

#### `toggleLike()` 实现逻辑

```java
@PostMapping("/{taskId}/like")
public AjaxResult toggleLike(@PathVariable Long taskId) {
    Long userId = VirtualSecurityUtils.getCurrentUserId();
    VirtualTaskLike existing = taskLikeMapper.selectByUserIdAndTaskId(userId, taskId);
    boolean liked;
    if (existing != null) {
        // 取消点赞
        taskLikeMapper.deleteByUserIdAndTaskId(userId, taskId);
        aiVideoTaskMapper.decrementLikeCount(taskId);
        liked = false;
    } else {
        // 点赞
        VirtualTaskLike like = new VirtualTaskLike();
        like.setUserId(userId); like.setTaskId(taskId);
        taskLikeMapper.insert(like);
        aiVideoTaskMapper.incrementLikeCount(taskId);
        liked = true;
    }
    AiVideoTask task = aiVideoTaskService.selectById(taskId);
    Map<String, Object> result = new HashMap<>();
    result.put("liked", liked);
    result.put("likeCount", task.getLikeCount());
    return AjaxResult.success(result);
```

### 4.0 签到积分奖励规则

> 源码位置：`VirtualCheckinServiceImpl.java` → `private static final int[] CHECKIN_CREDITS`

每日签到积分按**连续签到天数**（第 1-7 天循环）递增，第 7 天为大奖：

| 连续天数 | 奖励积分 | 备注 |
| --- | --- | --- |
| 第 1 天 | 3 积分 | 开始签到 |
| 第 2 天 | 5 积分 | |

**规则说明**：

- 每天 0 点刷新，同一自然日只能签到一次（`UNIQUE KEY uk_user_date`）

```java
// 签到积分规则（Day 1-7）
private static final int[] CHECKIN_CREDITS = { 3, 5, 10, 3, 3, 5, 50 };

@Override
public Map<String, Object> getCheckinStatus(Long userId) {
    String today = LocalDate.now().toString(); // "2026-02-26"
    VirtualDailyCheckin todayRecord = checkinMapper.selectTodayByUserId(userId, today);

    // 计算当前连续天序号
    int sequence = calculateCurrentSequence(userId, today);

    // 构造 7 天展示数据
    List<Map<String, Object>> days = new ArrayList<>();
    for (int i = 1; i <= 7; i++) {
        Map<String, Object> day = new HashMap<>();
        day.put("day", i);
        day.put("credits", CHECKIN_CREDITS[i - 1]);
        day.put("isGrandPrize", i == 7);
        // 已领取：i < sequence（之前天），或 i == sequence && todayClaimed
        day.put("claimed", i < sequence || (i == sequence && todayRecord != null));
        day.put("isToday", i == sequence);
        days.add(day);
    }

    Map<String, Object> result = new HashMap<>();
    result.put("todayClaimed", todayRecord != null);
    result.put("currentSequence", sequence);
    result.put("days", days);
    return result;
}

@Override
@Transactional
public int claimCheckin(Long userId) {
    String today = LocalDate.now().toString();
    // 防重复签到
    if (checkinMapper.selectTodayByUserId(userId, today) != null) {
        throw new ServiceException("今日已签到");
    }
    int sequence = calculateCurrentSequence(userId, today);
    int credits = CHECKIN_CREDITS[sequence - 1];

    // 插入签到记录
    VirtualDailyCheckin checkin = new VirtualDailyCheckin();
    checkin.setUserId(userId);
    checkin.setCheckinDate(Date.valueOf(LocalDate.now()));
    checkin.setDaySequence(sequence);
    checkin.setCreditsAwarded(credits);
    checkinMapper.insert(checkin);

    // 增加积分（复用已有积分服务）
    // addCredits 签名：(Long userId, Long amount, String txType, String businessType, Long businessId, String description)
    creditsService.addCredits(userId, (long) credits, "checkin", "daily_checkin", checkin.getCheckinId(), "Daily checkin Day" + sequence);
    return credits;
}

private int calculateCurrentSequence(Long userId, String today) {
    // 查询昨天是否已签到
    String yesterday = LocalDate.parse(today).minusDays(1).toString();
    VirtualDailyCheckin yesterdayRecord = checkinMapper.selectYesterdayByUserId(userId, yesterday);
    if (yesterdayRecord == null) {
        return 1; // 昨天未签到，重置为第1天
    }
    int nextSeq = yesterdayRecord.getDaySequence() + 1;
    return nextSeq > 7 ? 1 : nextSeq; // 第7天后循环回第1天
}
```

### 9.1 全局视频背景方案（AppLayout 级别，非逐页包裹）

> **架构决策**：视频背景放在 `AppLayout.vue` 全局级别，而非每个页面单独包裹 `<VideoBg>`。
>
> **原因**：若每个页面各自创建 `<video>` 元素，路由切换时视频会重新加载（白屏闪烁），
> 且多个 `<video>` 实例占用内存。全局放置只有一个视频实例，切换页面时背景连续不中断。
>
> `AppLayout.vue` 已有 `isFullBgPage` 计算属性（检测 Home/Showcase/Profile），可复用控制逻辑。

#### 修改 `src/components/layout/AppLayout.vue`

**当前代码**：

```vue
<script setup lang="ts">
const isHomePage = computed(() => route.path === '/')
const isShowcasePage = computed(() => route.path === '/showcase')
const isProfilePage = computed(() => route.path === '/profile')
```

```vue
  isHomePage.value || isShowcasePage.value || isProfilePage.value || isGeneratePage.value
)
</script>

  </div>
</template>

  position: fixed;
  top: 0; left: 0;
  width: 100%; height: 100%;

  pointer-events: none;
```

- `v-if="isFullBgPage"` 在非全屏页面时不创建 video 元素，节省资源

**为什么 i18n 提前到 Step 7**：所有前端组件都依赖 `t()` 调用，先补齐 key 可以避免后续每个步骤都回来改语言文件。

## 注意事项

### 功能安全

1. **未登录用户**：点赞、签到、头像/昵称编辑均需先登录，前端拦截并弹出登录 Modal
2. **积分并发安全**：`claimCheckin` 使用数据库唯一约束（`UNIQUE KEY uk_user_date`）防重复，服务层再做二次校验
3. **点赞原子性**：`like_count` 的增减使用 SQL 表达式（`like_count + 1`）而非先查后更新
4. **验证码安全**：Redis 验证码使用后立即删除，防止重放攻击；同时限制 60 秒发送间隔
5. **头像上传**：复用已有 pet-file 模块（`IFileService.upload()` 返回 `FileUploadResult`），与任务图片上传使用相同接口

### Security 配置确认

1. **邮箱登录端点**：`/api/virtual/auth/email/send-code` 和 `/api/virtual/auth/email/login` 位于 `/api/virtual/auth/**` 路径下，SecurityConfig 中已配置 `permitAll()`，**无需额外配置**
2. **签到端点**：`/api/virtual/checkin/**` 不在 SecurityConfig 白名单中，会被 `VirtualJwtAuthenticationFilter` 拦截，**必须携带 token 才能访问**，这是正确的
3. **点赞端点**：`POST /api/virtual/task/{taskId}/like` 不在白名单中（白名单仅有 `GET /task/options` 和 `GET /task/showcase`），JWT 拦截正确

### 架构决策

1. **Google OAuth**：本次实现完整的 Google OAuth 登录。后端已有 `GoogleOAuth2Provider`（授权URL生成+code换token+用户信息获取）和 `registerOrLoginByOAuth()`（自动注册+邮箱账号合并+积分初始化），只需在 `VirtualAuthController` 补全 2 个 HTTP 端点。前端采用弹窗模式（`window.open` + `postMessage`），新增 `OAuthCallback.vue` 处理回调。部署时需在 Google Cloud Console 配置 OAuth Client ID/Secret 和 redirect URI
2. **Showcase 表 vs Task 表的 like**：本次点赞功能基于 `ai_video_task` 表（因为 showcase 瀑布流数据源是 `/api/virtual/task/showcase`）。`VirtualShowcaseController` 中已有的 `/{showcaseId}/like` 端点（无防重复、无用户追踪）暂时保留但**不再由前端调用**
3. **Settings.vue 孤立页面**：`src/views/user/Settings.vue` 存在但未在 `router/index.ts` 中注册，且缺少 `settings.*` i18n key。本次不处理，后续单独迭代

### A. Google OAuth 配置

**适用功能**：Google 一键登录（第 1.5 步）

**申请地址**：[Google Cloud Console](https://console.cloud.google.com/) → APIs & Services → Credentials

**申请步骤**：

1. 进入 Google Cloud Console，创建或选择项目
2. 进入 **APIs & Services → Credentials**，点击 **Create Credentials → OAuth 2.0 Client ID**
3. 应用类型选择 **Web application**
4. **Authorized JavaScript origins** 添加：
   - 开发：`http://localhost:3000`
   - 生产：`https://yourdomain.com`
5. **Authorized redirect URIs** 添加：
   - 开发：`http://localhost:3000/auth/callback`
   - 生产：`https://yourdomain.com/auth/callback`
6. 创建完成后，记录 **Client ID** 和 **Client Secret**
7. 启用 **Google People API**（用于获取用户头像）

**配置项**（填入 `application-pet.yml` 或设为环境变量）：

```yaml
pet:
  virtual:
    oauth2:
      google:
        client-id: ${GOOGLE_CLIENT_ID:}           # 填入 Client ID
        client-secret: ${GOOGLE_CLIENT_SECRET:}   # 填入 Client Secret
        redirect-uri: ${GOOGLE_REDIRECT_URI:http://localhost:3000/auth/callback}
```

### B. 邮件服务（SMTP）配置

**适用功能**：邮箱验证码登录（第一步）

**推荐服务**：Gmail SMTP（免费）、SendGrid（大量发送）、Amazon SES（低成本）

**Gmail SMTP 申请步骤**：

1. 进入 Google 账号 → 安全性 → 两步验证（必须先启用）
2. 在 **应用专用密码** 处生成一个 16 位密码（用作 `MAIL_PASSWORD`）
3. SMTP 主机：`smtp.gmail.com`，端口：`587`（TLS）或 `465`（SSL）

**配置项**（填入 `application-pet.yml` 或设为环境变量）：

```yaml
spring:
  mail:
    host: ${MAIL_HOST:smtp.gmail.com}
    port: ${MAIL_PORT:587}
    username: ${MAIL_USERNAME:your-email@gmail.com}
    password: ${MAIL_PASSWORD:your-app-password}
    properties:
      mail.smtp.auth: true
      mail.smtp.starttls.enable: true
```

> **注意**：`MAIL_PASSWORD` 是应用专用密码（16位），不是 Gmail 账号密码。

### E. 文件存储（S3 / Cloudflare R2）配置

**适用功能**：用户头像、视频文件上传

**推荐**：Cloudflare R2（免费额度大，兼容 S3 API）或 AWS S3

**Cloudflare R2 申请步骤**：

1. 进入 [Cloudflare Dashboard](https://dash.cloudflare.com/) → R2 → 创建存储桶
2. 进入 **管理 R2 API 令牌** → 创建令牌（权限：对象读写）
3. 记录 Access Key ID 和 Secret Access Key
4. Endpoint 格式：`https://<accountId>.r2.cloudflarestorage.com`

**配置项**：

```yaml
pet:
  file:
    s3:
      access-key-id: ${S3_ACCESS_KEY_ID:}           # R2/S3 Access Key
      access-key-secret: ${S3_ACCESS_KEY_SECRET:}   # R2/S3 Secret Key
      endpoint: ${S3_ENDPOINT:}                     # R2: https://<id>.r2.cloudflarestorage.com
      region: ${S3_REGION:auto}                     # R2 固定填 auto，S3 填对应区域
      bucket: ${S3_BUCKET:pet-platform}             # 存储桶名称
      public-domain: ${S3_PUBLIC_DOMAIN:}           # 公共访问域名（R2 自定义域或 CDN）
```
